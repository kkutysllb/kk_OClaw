# OClaw 封面背景升级设计

## 概述

本设计面向 `assets/cover.svg` 的背景升级，目标是在不改动现有标题层级、文案布局和信息结构的前提下，把当前偏干净理性的蓝色科技背景，提升为以“中心星云能量场”为核心、辅以轻量 HUD 点缀的未来感封面。

本次升级重点不是推翻重做，而是在现有视觉基础上增强空间纵深、发光层次和画面气场，让整体更有“科技炫酷”的第一眼冲击力，同时保持高级、克制、可读。

## 目标

- 提升封面背景的科技感、未来感和高级感。
- 保持 `OClaw` 仍然是第一视觉焦点。
- 保留当前文案、版式和信息层级。
- 以最小、低风险的 SVG 改动完成背景增强。
- 确保生成的 SVG 仍然符合标准、可正常打开和渲染。

## 非目标

- 不整张封面推倒重做。
- 不修改主标题、副标题、功能标签和底部技术栈文案。
- 不引入高饱和、多彩霓虹风，避免破坏当前冷静的品牌气质。
- 不加入动画或强依赖特定浏览器行为的 SVG 特性。

## 当前状态分析

当前封面已经具备基础科技风元素：

- 深蓝色渐变背景
- 斜向光束
- 小粒子漂浮
- 标题发光效果
- 功能标签和底部技术栈信息

当前主要问题不在于元素太少，而在于背景的视觉叙事偏平，主要依赖几条斜向光束来提供“动势”。这种方式虽然整洁，但缺少围绕标题建立起来的中心能量感和空间层次，因此整体更像“通用科技背景”，还不够炫酷和有记忆点。

## 推荐方向

采用“星云核心增强 + 轻量 HUD 点缀”的方向：

- 视觉能量从当前的“斜线穿场”转为“围绕标题聚能”。
- 画面增加前景、中景、背景三层空间关系。
- HUD 元素只作为边缘细节点缀，不能抢标题。
- 配色继续使用蓝、青、靛一类冷色范围，延续当前基调。

这个方向的优势是：在不破坏现有版式的情况下，可以显著提高视觉冲击力，同时仍保留平台型产品封面的稳定感和专业感。

## 视觉系统设计

### 1. 底层深空背景

将现有背景升级为更有纵深感的深空底层：

- 保留深蓝色作为整体基底。
- 在标题区域后方加入柔和的蓝紫色径向辉光。
- 四角和边缘略微压暗，以突出画面中心。

预期效果：

- 画面不再只是平面渐变，而是有更明显的空间包裹感。
- 标题区域会天然形成视觉焦点，为后续星云和能量层做承托。

### 2. 中层星云雾面

在标题周围加入 2 到 3 层柔和的星云型光雾：

- 形态保持抽象、矢量化，不追求照片式宇宙效果。
- 通过模糊椭圆、径向渐变、半透明叠层来构成。
- 中心区域亮度更集中，向外自然散开。

预期效果：

- 形成“AI 能量场”式的中心氛围。
- 在不增加杂乱感的前提下显著提升画面层次和质感。

### 3. 能量环与弧形轨迹

在标题周边增加很淡的弧线、半环和轨迹线：

- 元素要偏轻，更多是暗示“聚能”和“运转”。
- 不让明显硬线直接穿过标题文字。
- 形态以环绕、包围、收束为主，不做夸张爆炸式表现。

预期效果：

- 让画面从“几条光束扫过”升级为“围绕品牌聚焦”的视觉叙事。
- 增强科技系统感和设计完成度。

### 4. 粒子系统增强

保留当前粒子语言，但重建层次：

- 将粒子拆分为背景粒子、中层粒子、前景高亮粒子。
- 更明确地区分大小、透明度和密度。
- 增加极少量短拖尾粒子，营造轻微运动感。
- 总量仍然要克制，避免粒子过多导致噪声感。

预期效果：

- 粒子不再像均匀撒点，而像真正漂浮在星云场中。
- 画面会更有活性，但不会变成喧闹的特效堆叠。

### 5. 轻量 HUD 边缘点缀

只在边缘区域加入非常淡的 HUD 风格元素：

- 可使用少量圆弧刻度、短扫描线、角落框线或分段轨迹。
- 避开副标题和下方功能标签区域。
- 透明度保持很低，只作为“精致科技细节”存在。

预期效果：

- 背景更明确地传达“AI 平台”而非泛化的星空图。
- 系统感得到增强，但不会破坏整体高级感。

## 现有元素调整策略

### 斜向光束

现有斜向光束不需要完全删除，但需要降权：

- 最多保留一组较强的辅助性光束。
- 整体透明度和存在感下调。
- 不再让斜线成为主叙事中心。

### 标题周围辉光

标题本身的发光可以保留，但要把更多气场交给背景承担：

- 标题发光负责保证可读性。
- 标题背后的背景辉光负责营造电影感和能量感。
- 亮度整体受控，避免把文字冲淡。

## 构图约束

- 主标题仍然必须是最亮、最清晰的中心内容。
- 副标题要保持一眼可读。
- 功能标签和底部技术栈文字不能被背景干扰。
- 视觉中心仍保持在当前标题区域附近。
- 升级不引起整体布局位移。

## 技术约束

- 继续使用标准 SVG 图形、渐变、滤镜和描边能力。
- 避免高风险、兼容性差的 SVG 特性。
- 文件结构保持清晰，便于后续维护和排错。
- 修改完成后必须重新进行 XML/SVG 解析校验。

## 实施范围

本次实施范围应保持收敛：

- 按需补充少量新的 `defs` 渐变和滤镜定义。
- 优先替换或弱化现有背景元素，而不是无限叠加新层。
- 新增少量分组元素用于星云、能量弧和边缘 HUD 点缀。
- 不改动文字和整体版式结构。

## 验收标准

完成后至少满足以下标准：

- SVG 文件可以被正常打开，无解析报错。
- `OClaw` 标题在第一眼仍然清晰突出。
- 背景相比当前版本更有纵深、更高级、更未来感。
- 画面整体平衡，不显拥挤或凌乱。
- 底部辅助信息在背景增强后仍保持可读。

## 风险与应对

### 风险一：背景过于复杂

应对方式：

- HUD 元素只放在边缘区域。
- 控制高亮粒子和能量环数量。
- 主动降低旧光束的存在感。

### 风险二：标题可读性下降

应对方式：

- 控制标题正后方的最亮区域范围和强度。
- 避免明显线条穿过标题。
- 用背景辉光衬托标题，而不是压过标题。

### 风险三：风格变成廉价通用科幻图

应对方式：

- 保持当前冷色、克制的配色逻辑。
- 以优雅的辉光层次取代堆砌式界面装饰。
- 控制元素数量，留出足够呼吸感。

## 最终建议

建议按“中心星云能量场 + 分层粒子 + 边缘轻 HUD 点缀”的组合实施。这是当前风险最低、收益最高的升级方案，既能明显提升背景图的科技炫酷感，也能保持现有封面结构、品牌气质和信息可读性。
