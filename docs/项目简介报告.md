# KKOCLAW 项目简介报告

> 开源超级智能体运行时平台 — 让 AI 真正拥有执行能力

---

## 一、项目定位

KKOCLAW 是一个开源的**超级智能体运行时（Super Agent Harness）**，基于 LangGraph + LangChain 构建。它将子智能体（Sub-Agents）、长期记忆（Memory）、沙箱执行环境（Sandbox）和可扩展技能系统（Skills）深度整合，使 AI 智能体能够自主完成复杂、多步骤的实际任务。

**核心价值**：开箱即用、高度可扩展、支持多模型多渠道，一个平台覆盖从研究分析到内容生成的全场景 AI 执行需求。

## 二、系统架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                    客户端 / 浏览器 / IM 渠道                    │
└──────────────────────────┬──────────────────────────────────┘
                           │
                    ┌──────▼──────┐
                    │  Nginx :9191 │
                    └──┬──────┬───┘
              /api/*     │      │  /*
         ┌──────────────▼┐  ┌──▼──────────┐
         │  Gateway :9193 │  │ Frontend     │
         │  (FastAPI)     │  │ (Next.js)    │
         └───────┬────────┘  │ :9192        │
                 │            └──────────────┘
     ┌───────────▼────────────┐
     │  内嵌 LangGraph 运行时    │
     │  ┌─────────────────┐   │
     │  │   Lead Agent     │   │
     │  │  18层中间件链      │   │
     │  │  ┌─────┐ ┌─────┐│   │
     │  │  │子智能│ │工具系││   │
     │  │  │体系统│ │统    ││   │
     │  │  └──┬──┘ └──┬──┘│   │
     │  │     │       │    │   │
     │  │  ┌──▼──┐ ┌──▼──┐ │   │
     │  │  │Sandbox│ │MCP │ │   │
     │  │  │执行环境│ │工具 │ │   │
     │  │  └─────┘ └─────┘ │   │
     │  └─────────────────┘   │
     └────────────────────────┘
```

**关键设计决策**：
- Gateway 内嵌 LangGraph 运行时（非独立进程），零网络开销
- 18 层中间件链处理所有横切关注点（安全、审计、限流、记忆等）
- 沙箱系统支持本地 / Docker / K8s 三种隔离模式
- 双线程池（调度池 + 执行池）驱动子智能体并行执行

## 三、核心能力矩阵

### 3.1 智能体系统

| 能力 | 说明 |
|------|------|
| **Lead Agent** | 主智能体，负责理解意图、拆解任务、调度子智能体、综合结果 |
| **Sub-Agents** | 动态并行子智能体，最多 3 个并发，15 分钟超时 |
| **自定义 Agent** | 支持通过配置定义专用子智能体，可自定义提示词、工具白名单、独立模型 |
| **SSE 流式** | 实时事件流：task_started → task_running → task_completed/failed |

### 3.2 沙箱执行环境

| 模式 | 适用场景 | 隔离级别 |
|------|----------|----------|
| **LocalSandbox** | 本地开发 | 进程级 |
| **Docker Sandbox** | 生产部署 | 容器级 |
| **K8s Sandbox** | 多租户 | Pod 级 |

沙箱提供 7 个核心工具：`bash`、`ls`、`read_file`、`write_file`、`str_replace`、`glob`、`grep`，并自动进行虚拟路径 ↔ 物理路径映射。

### 3.3 技能系统（25 个内置技能）

| 类别 | 技能 |
|------|------|
| **研究分析** | deep-research, github-deep-research, consulting-analysis, analysis-report |
| **内容生成** | ppt-generation, podcast-generation, newsletter-generation, xlsx-creator |
| **视觉创作** | image-generation, video-generation, music-generation, frontend-design |
| **学术工具** | academic-paper-review, systematic-literature-review |
| **开发工具** | code-documentation, bootstrap, skill-creator, find-skills |
| **数据处理** | data-analysis, chart-visualization, pdf-processing |
| **Web 工具** | web-design-guidelines, vercel-deploy-claimable, claude-to-kkoclaw, surprise-me |

技能采用 Markdown 格式（SKILL.md），按需渐进加载，保持上下文窗口精简。

### 3.4 记忆系统

LLM 驱动的跨会话持久化记忆：
- **自动提取**：从对话中提取用户偏好、知识背景、工作习惯
- **置信度评分**：每个事实带 0-1 置信度，低于阈值自动淘汰
- **防抖写入**：30 秒防抖 + 原子性文件写入，避免频繁 LLM 调用
- **按用户隔离**：每个用户独立的 `memory.json`

### 3.5 Token 用量监控

| 统计维度 | 说明 |
|----------|------|
| **任务次数** | 每次对话的执行次数 |
| **API 调用次数** | 实际 LLM 模型调用次数（含子智能体、中间件） |
| **Token 消耗** | 按输入/输出/总计，支持按模型、按调用方拆分 |
| **时序趋势** | 日粒度的 Token 消耗趋势图，支持月度筛选 |
| **5 分钟自动刷新** | 后台静默刷新，不影响用户操作 |

### 3.6 IM 渠道集成

支持 5 大即时通讯平台，**无需公网 IP**：

| 渠道 | 传输方式 | 特点 |
|------|----------|------|
| Telegram | Bot API 长轮询 | 简单部署 |
| Slack | Socket Mode | 中等复杂度 |
| 飞书/Lark | WebSocket 长连接 | 流式卡片更新 |
| 企业微信 | WebSocket | 智能机器人 |
| 钉钉 | Stream Push | AI 卡片流式 |

## 七、安全设计

| 层面 | 措施 |
|------|------|
| **沙箱隔离** | Docker/K8s 容器级隔离，文件操作防路径穿越 |

| **密钥安全** | API Key 通过环境变量引用，不存储在配置文件 |

## 八、中间件体系（18 层）

按严格顺序执行，覆盖所有横切关注点：

```
ThreadData → Uploads → Sandbox → DanglingToolCall → LLMErrorHandling
→ Guardrail → SandboxAudit → ToolErrorHandling → Summarization
→ TodoList → TokenUsage → Title → Memory → ViewImage
→ DeferredToolFilter → SubagentLimit → LoopDetection → Clarification
```

关键中间件说明：
- **LoopDetectionMiddleware**：三层循环防护（工具频率 → 连续相同调用 → 强制终止 + 状态清理）
- **LLMErrorHandlingMiddleware**：区分瞬时限流（429 retry）与硬配额耗尽（insufficient_quota），支持 Google retryDelay 解析
- **TokenUsageMiddleware**：按模型 + 调用方（lead_agent / subagent / middleware）追踪 Token
- **SummarizationMiddleware**：上下文超限时自动压缩，保留近期技能调用
- **MemoryMiddleware**：异步记忆提取，30 秒防抖

## 十二、关键数据流

### 请求处理

```
用户消息 → Nginx → Gateway → LangGraph 运行时
  → ThreadState 加载 → 18 层中间件处理
  → Model 调用（含 Tool 选择）
  → Sandbox 执行 → SSE 流式返回
```

### 子智能体执行

```
Lead Agent 识别并行任务
  → 调度池（3 worker）接收
  → 执行池（3 worker）运行
  → 各子智能体独立 Sandbox + Context
  → 结果合并 → 返回 Lead Agent
```

### 记忆更新

```
对话结束 → MemoryMiddleware 过滤
  → 30s 防抖入队 → LLM 提取事实
  → 置信度评估 → 原子写入 memory.json
  → 下次对话注入 Top 15 事实
```

## 十三、内嵌 Python Client

无需启动 HTTP 服务即可使用：

```python
from kkoclaw.client import KKOCLAWClient

client = KKOCLAWClient()

# 对话
response = client.chat("分析这篇论文", thread_id="my-thread")

# 流式响应
for event in client.stream("你好"):
    if event.type == "messages-tuple" and event.data.get("type") == "ai":
        print(event.data["content"])

# 管理操作
models = client.list_models()
skills = client.list_skills()
client.upload_files("thread-1", ["./report.pdf"])
```
