# KKOCLAW 安装指南

本文件供编程智能体使用。如果 KKOCLAW 仓库尚未克隆并打开，请先克隆仓库，然后从仓库根目录继续操作。

## 目标

以风险最低的方式在用户机器上搭建 KKOCLAW 本地开发工作区。

默认偏好：

1. Docker 开发环境
2. 本地开发环境

不要假设 API 密钥或模型凭证已存在。设置好所有可以安全准备的内容，然后停止，简要总结用户仍需提供的内容。

## 操作规则

- 保持幂等性。重新运行本文档不应破坏已有的设置。
- 优先使用现有仓库命令，而不是临时编写的 shell 命令。
- 未经用户明确同意，不要使用 `sudo` 或安装系统软件包。
- 除非用户要求，否则不要覆盖已有的用户配置值。
- 如果某个步骤失败，停止，解释障碍，并提供最小的下一步行动。
- 如果存在多种设置路径，当 Docker 已可用时优先选择 Docker。

## 成功标准

当满足以下所有条件时，视为设置成功：

- KKOCLAW 仓库已克隆，当前工作目录为仓库根目录。
- `config.yaml` 已存在。
- 对于 Docker 设置，`make docker-init` 已成功完成，Docker 前置条件已准备好，但尚未假设服务正在运行。
- 对于本地设置，`make check` 已通过或报告没有缺失的前置依赖，且 `make install` 已成功完成。
- 用户收到了启动 KKOCLAW 的确切下一步命令。
- 用户还收到了 `config.yaml` 中任何缺失的模型配置或引用的环境变量名称，而不检查包含密钥的文件的实际值。

## 步骤

- 如果当前目录不是 KKOCLAW 仓库根目录，则先克隆仓库，然后切换到仓库根目录。
- 检查 `Makefile`、`backend/`、`frontend/` 和 `config.example.yaml` 是否存在，以确认当前目录是 KKOCLAW 仓库根目录。
- 检测 `config.yaml` 是否已存在。
- 如果 `config.yaml` 不存在，运行 `make config`。
- 检测 Docker 是否可用，以及守护进程是否可以通过 `docker info` 访问。
- 如果 Docker 可用：
  - 运行 `make docker-init`。
  - 这仅视为 Docker 前置条件准备。不要声称应用服务、compose 验证或镜像构建已经成功。
  - 除非用户明确要求或设置请求明确包含启动验证，否则不要启动长时间运行的服务。
  - 告诉用户推荐的下一步命令是 `make docker-start`。
- 如果 Docker 不可用：
  - 运行 `make check`。
  - 如果 `make check` 报告缺失系统依赖（如 `node`、`pnpm`、`uv` 或 `nginx`），停止并报告缺失的工具，而不是尝试特权安装。
  - 如果前置条件满足，运行 `make install`。
  - 告诉用户推荐的下一步命令是 `make dev`。
- 仅检查 `config.yaml` 中缺失的模型条目或引用的环境变量占位符。不要读取 `.env`、`frontend/.env` 或其他包含密钥的文件。
- 如果没有配置模型，告诉用户必须在 `config.yaml` 的 `models` 下至少添加一个条目。
- 如果 `config.yaml` 引用了如 `$OPENAI_API_KEY` 这样的变量，告诉用户哪些变量名仍需要真实值，但不要通过打开包含密钥的文件来验证它们。
- 如果仓库看起来已经配置好了，避免重复昂贵的工作，除非验证环境是必要的。

## 验证

使用与所选设置路径匹配的最轻量验证。

对于 Docker 设置：

- 确认 `make docker-init` 已完成成功。
- 确认 `config.yaml` 已存在。
- 明确声明 Docker 服务尚未启动，`make docker-start` 仍然是第一个真正的启动步骤。
- 除非用户要求，否则不要将后台服务保持运行状态。

对于本地设置：

- 确认 `make install` 已完成成功。
- 确认 `config.yaml` 已存在。
- 除非用户要求，否则不要将后台服务保持运行状态。

## 最终响应格式

返回简短的报告，包含：

1. 使用的设置路径：Docker 或本地
2. 达到的设置级别：Docker 前置条件已准备或本地依赖已安装
3. 创建或检测到的文件：例如 `config.yaml`
4. 用户仍需执行的操作：模型配置、环境变量值、认证文件或无
5. 启动 KKOCLAW 的确切下一步命令

## 立即执行

完成以上步骤。完成后，在设置边界处停止并报告状态，而不是继续进入无关的项目工作。
