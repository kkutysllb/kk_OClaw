# QA Test Plan Skill

## Purpose

Define how the project will be validated before release.

## Workflow

1. Map acceptance criteria to automated and manual checks.
2. Cover happy path, edge cases, permissions, error states, and recovery.
3. Include browser/device coverage for UI projects.
4. Identify smoke tests for release readiness.
5. Keep the plan practical enough to execute.

## Review Checklist

- Every release blocker has a verification step.
- Manual checks are written as clear actions and expected results.
- Existing automation is reused where possible.
- Risky areas get deeper coverage.
