# Requirements Analysis Skill

## Purpose

Turn a broad idea into clear, buildable requirements.

## Workflow

1. Identify the target users and their primary jobs.
2. Separate goals, constraints, assumptions, and non-goals.
3. Capture core workflows and important edge cases.
4. Resolve ambiguity before architecture or implementation.
5. Produce requirements that can be tested through acceptance criteria.

## Review Checklist

- Requirements are behavior-oriented, not implementation guesses.
- Non-goals prevent scope creep.
- Unknowns are explicit.
- User-visible success can be verified.
