# Handoff Docs Skill

## Purpose

Make completed work understandable and maintainable by someone else.

## Workflow

1. Summarize what was built and why.
2. List key files, architecture boundaries, and runtime assumptions.
3. Document commands for setup, test, build, and deploy.
4. Capture known limitations and follow-up work.
5. Link verification evidence and release notes.

## Review Checklist

- Handoff docs do not require reading the full commit diff first.
- Next steps are actionable and prioritized.
- Known risks are honest and specific.
- The document matches the final implementation.
