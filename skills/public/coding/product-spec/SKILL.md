# Product Spec Skill

## Purpose

Create a concise product artifact that guides design and implementation.

## Workflow

1. State the problem and intended users.
2. Define MVP scope and out-of-scope items.
3. Describe primary user flows and expected states.
4. Capture constraints, risks, and dependencies.
5. Link each requirement to an acceptance criterion.

## Review Checklist

- The spec is short enough to guide execution.
- It avoids premature technical decisions unless required by constraints.
- Edge states such as empty, loading, failure, and permissions are included.
- Success metrics or observable outcomes are listed.
