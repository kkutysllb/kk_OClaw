# Technical Design Skill

## Purpose

Define how the system will be built before editing implementation files.

## Workflow

1. Map requirements to components and boundaries.
2. Define data models, API contracts, and integration points.
3. Identify failure modes, security concerns, and migration needs.
4. Choose the simplest design that satisfies the acceptance criteria.
5. List verification commands and test coverage expected from the design.

## Review Checklist

- The design follows existing project patterns.
- Dependencies flow in the right direction.
- Persistence and runtime state boundaries are explicit.
- Tradeoffs and rejected options are recorded when important.
