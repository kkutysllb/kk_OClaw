# Planning Skill

## Purpose

Turn broad coding goals into ordered, verifiable steps.

## Workflow

1. State the objective and non-goals.
2. Identify affected layers and files.
3. Split the work into phases that can be tested independently.
4. Put high-risk or dependency-unblocking work first.
5. Keep unresolved frontend polish or follow-up items visible without blocking
   backend/core completion.

## Review Checklist

- Each phase has a concrete output.
- The plan separates backend engine, API, frontend UI, and tests when relevant.
- The plan avoids unrelated refactors.
- Verification commands are attached to the phase they validate.
