# Acceptance Criteria Skill

## Purpose

Convert requirements into objective checks that determine whether work is done.

## Workflow

1. Write criteria from the user's perspective.
2. Include happy path, validation errors, permissions, and empty states.
3. Map criteria to tests or manual verification steps.
4. Keep criteria measurable and unambiguous.
5. Update criteria when scope changes.

## Review Checklist

- Every core requirement has at least one criterion.
- Criteria can fail clearly.
- Nonfunctional requirements such as performance or security are included when
  relevant.
- Release blockers are distinguished from nice-to-have polish.
