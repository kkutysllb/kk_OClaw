# Operations Runbook Skill

## Purpose

Give maintainers clear instructions for running and supporting the system.

## Workflow

1. Document startup, shutdown, health checks, and logs.
2. List common failure modes and response steps.
3. Capture backup, restore, migration, and rollback procedures.
4. Include ownership and escalation notes when known.
5. Keep commands copyable and environment assumptions explicit.

## Review Checklist

- Runbook can be used during an incident.
- Commands include working directory and required variables.
- Recovery steps avoid data loss.
- Monitoring and alert signals are explained.
